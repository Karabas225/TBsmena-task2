Техническое задание / Описание функциональных характеристик Программного обеспечения "Т-Библиотека"

1. Предмет

1.1. Настоящий документ определяет функциональные характеристики, технические требования и состав программного обеспечения "Т-Библиотека" (далее - ПО), предназначенного для автоматизации учета личной (домашней) библиотеки пользователя.

2. Функциональные характеристики

2.1. ПО функционирует в режиме интерфейса командной строки (консольное приложение) и обеспечивает выполнение следующих операций с электронным каталогом (библиотекой):

2.1.1. Создание учетных записей книг: Обеспечивается возможность внесения в каталог новой книги с обязательным указанием следующих атрибутов: название, автор, жанр, год выпуска, краткое описание (аннотация). Каждой записи автоматически присваивается уникальный идентификационный номер (ID).

2.1.2. Отображение каталога: Реализован вывод перечня всех имеющихся записей в формате таблицы с возможностью:
* Фильтрации данных по жанру и по статусу прочтения.
* Сортировки данных по названию, автору или году выпуска.

2.1.3. Управление статусом прочтения: Предусмотрена возможность изменения бинарного статуса книги ("прочитана" / "не прочитана").

2.1.4. Управление списком избранного:
* Реализована функция добавления книги в список "Избранное" (маркировка специальным символом).
* Реализована функция отображения исключительно книг, включенных в список "Избранное".
* Реализована функция удаления книги из списка "Избранное".
* Система предотвращает повторное добавление одной и той же книги в избранное.

2.1.5. Поиск: Реализован поиск записей по вхождению заданной строки в поля "Название", "Автор" и "Описание".

2.1.6. Удаление записей: Предусмотрена возможность удаления книги из каталога по ее уникальному идентификатору (ID).

2.1.7. Сохранение данных:
* Все изменения каталога автоматически фиксируются в файле внешнего хранения.
* При последующем запуске ПО производится автоматическая загрузка данных из указанного файла, что обеспечивает целостность информации между сеансами работы.

2.1.8. Защита от некорректных действий: Логика ПО исключает двойное применение идентичных команд к одной записи (например, попытка установить уже установленный статус прочтения).

3. Технические требования и стек технологий

3.1. ПО разработано на языке программирования Python версии 3.

3.2. Для обеспечения форматированного вывода данных (таблицы, панели, цветовое оформление) используется библиотека Rich.

3.3. Формат хранения данных - JSON.

4. Состав и структура поставки

4.1. ПО распространяется в виде набора файлов, организованных в следующую структуру каталогов (репозиторий):


```
t-library/
|---- src/                          # Исходный код
|   |---- models.py                 # Класс Book — как устроена книга
|   |---- storage.py                # Загрузка и сохранение JSON
|   |---- library.py                # Основная логика (добавление, удаление, поиск)
|   |---- ui.py                     # Всё, что связано с интерфейсом (меню, таблицы)
|   |---- utils.py                  # Вспомогательные функции (очистка экрана и т.п.)
|---- data/                          # Папка для файла с книгами
|   |---- library.json               # Файл, куда сохраняются книги
|---- requirements.txt                # Список необходимых библиотек
|---- README.md                       # Описание проекта
|---- READMEofficial.md
|---- main.py                         # Точка входа: запуск программы
```

**5. Требования к взаимодействию с пользователем**

5.1. Управление ПО осуществляется посредством ввода цифровых команд (пунктов меню). Интерфейс предусматривает наличие текстовых подсказок на каждом этапе взаимодействия, что обеспечивает возможность использования ПО пользователем, обладающим базовыми навыками работы в командной строке.
